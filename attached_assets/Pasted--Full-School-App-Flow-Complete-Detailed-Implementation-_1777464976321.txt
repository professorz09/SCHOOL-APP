# Full School App Flow – Complete Detailed Implementation Plan

## Context

The EduGrow school management app has a solid foundation: Supabase backend with 66 DB tables, RLS, auth, and most role-based UI screens built. However, several critical features and edge cases need implementation.

**Key Findings from Code Audit:**
- Document uploads are **fake** (local state only, never saved to Supabase storage)
- Staff salary has **no pro-rata logic** (full salary even for mid-month join/leave)
- Attendance **doesn't feed into salary** (two separate systems)
- Transport has **no date tracking** (no history of mid-year vehicle changes)
- Fee schedule generation RPC exists **but is never called** from UI
- Year closing has **auto-promotion logic** (which spec says "NO complex auto promotion")
- Roll numbers have **no uniqueness constraint** in DB
- 12th class **streams not persisted** (UI state only)
- Late fees have **schema but no calculation logic**

### 2. DOCUMENT UPLOAD & STORAGE

**Current Problem:** Upload UI is fake (just toggles local state). Never calls Supabase storage.

**Required Real Implementation:**

#### 2a. Student Documents Real Upload
```typescript
// In StudentsManager.tsx handleDocumentUpload()

const file = e.target.files?.[0];
if (!file) return;

// 1. Validate
if (file.size > 5*1024*1024) { error('Max 5MB'); return; }
if (!['image/jpeg', 'image/png', 'application/pdf'].includes(file.type)) {
  error('Only JPG, PNG, PDF'); return;
}

// 2. Upload to Supabase Storage
const filename = `${Date.now()}-${file.name}`;
const { data, error: uploadError } = 
  await supabase.storage
    .from('student-documents')
    .upload(`${schoolId}/${studentId}/${docType}/${filename}`, file);

if (uploadError) { error(uploadError.message); return; }

// 3. Get signed URL (valid 24 hours)
const { data: { publicUrl } } = 
  supabase.storage
    .from('student-documents')
    .getPublicUrl(`${schoolId}/${studentId}/${docType}/${filename}`);

// 4. Save to student_documents table
await supabase.from('student_documents').insert({
  student_id: studentId,
  doc_type: docType,
  doc_url: publicUrl,
  uploaded_by: userId,
  uploaded_at: new Date()
});

// 5. UI update
setDocuments(prev => prev.map(d => 
  d.type === docType 
    ? { ...d, uploaded: true, url: publicUrl } 
    : d
));
```

#### 3b. Only ONE Active Year Enforcement

```sql
-- Trigger: When is_active = TRUE, set all others to FALSE
CREATE OR REPLACE FUNCTION enforce_single_active_year()
RETURNS TRIGGER AS $$
BEGIN
  IF NEW.is_active = TRUE THEN
    UPDATE academic_years 
    SET is_active = FALSE 
    WHERE school_id = NEW.school_id 
      AND id != NEW.id 
      AND is_active = TRUE;
  END IF;
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER trigger_single_active_year
AFTER UPDATE OF is_active ON academic_years
FOR EACH ROW
EXECUTE FUNCTION enforce_single_active_year();
```

#### 5b. Roll Number Auto-Suggest & Uniqueness

```sql
-- DB Constraint
ALTER TABLE student_academic_records
  ADD CONSTRAINT uq_roll_per_section_year
  UNIQUE(academic_year_id, section_id, roll_number);

-- UI Logic (TypeScript)
const suggestRollNumber = async (academicYearId, sectionId) => {
  const { data } = await supabase
    .from('student_academic_records')
    .select('roll_number')
    .eq('academic_year_id', academicYearId)
    .eq('section_id', sectionId)
    .order('roll_number', { ascending: false })
    .limit(1);
  
  return data?.[0]?.roll_number ? data[0].roll_number + 1 : 1;
};

// Manual override with conflict check
const validateRollNumber = async (academicYearId, sectionId, rollNumber) => {
  const { data } = await supabase
    .from('student_academic_records')
    .select('student_id')
    .eq('academic_year_id', academicYearId)
    .eq('section_id', sectionId)
    .eq('roll_number', rollNumber);
  
  if (data?.length > 0) {
    return { conflict: true, conflictStudent: data[0].student_id };
  }
  return { conflict: false };
};
```

#### 6b. Fee Schedule Auto-Generation on Class Assignment

```
When student assigned to class (Section 5a, Step 2):

Call RPC: generate_student_fee_schedule(
  p_student_id,
  p_academic_year_id,
  p_fee_structure_id,
  p_discount_amount,  -- Fixed amount (₹500) or percentage (10%)
  p_is_rte            -- RTE student (govt pays, not parent)
)

RPC Logic:
  1. Delete all UNPAID installments for (student, year)
  2. For each fee head in fee_structure:
       frequency = MONTHLY (default)
       IF frequency == MONTHLY:
         FOR month IN Apr-Mar:
           amount = head.amount - (discount if monthly, else 0)
           due_date = monthly_due_dates[month]
           payer_type = GOVERNMENT if is_rte else PARENT
           fee_type = detect from head.name (TUITION, TRANSPORT, etc)
           
           INSERT INTO fee_installments(
             student_id, academic_year_id, month, due_date, 
             fee_type, amount, payer_type, status='UNPAID'
           )
       
       IF frequency == ANNUAL:
         amount = head.amount - discount
         due_date = first_month_date
         INSERT INTO fee_installments(...)
```

#### 6d. Payment Allocation (Oldest Due First)

```
Student owes:
  2026-27: ₹3,000 (previous year, overdue)
  2027-28 April: ₹5,000 (current, due 15 Apr)
  2027-28 May: ₹5,000 (current, due 15 May)

Allocation (oldest first):
  2026-27: ₹3,000 (PAID)
  2027-28 April: ₹5,000 (PAID)
  2027-28 May: ₹0 (not yet paid, but ₹0 allocation)
  BALANCE OWED: ₹5,000

RPC: record_fee_payment(
  p_student_id, p_amount, p_method, p_transaction_id
)

Logic:
  1. Get all UNPAID installments for student's ALL years, order by due_date ASC
  2. Loop through, allocate amount:
       remaining = 8000
       FOR inst IN unpaid_installments (oldest first):
         IF remaining >= inst.amount:
           inst.paid_amount = inst.amount
           remaining -= inst.amount
           inst.status = 'PAID'
         ELSE:
           inst.paid_amount = remaining
           remaining = 0
           inst.status = 'PARTIAL'
           BREAK
  3. IF remaining > 0:
       INSERT INTO advance_balances(
         student_id, amount = remaining
       )
       [This credit used for future payments]
  4. INSERT INTO payment_records(
       student_id, academic_year_id, amount, method, date
     )
```

#### 9b. RPC: record_school_payment (Oldest-Due-First)

```sql
CREATE OR REPLACE FUNCTION record_school_payment(
  p_school_id UUID,
  p_amount DECIMAL,
  p_transaction_id TEXT,
  p_method TEXT,  -- 'UPI', 'NEFT', 'CHEQUE', 'CASH'
  p_notes TEXT
)
RETURNS TABLE (year_id UUID, allocated_amount DECIMAL) AS $$
DECLARE
  v_remaining DECIMAL := p_amount;
  v_year RECORD;
  v_allocated DECIMAL;
  v_payment_id UUID;
BEGIN
  -- Record payment
  INSERT INTO school_payments(school_id, amount, txn_id, method, notes, paid_at)
  VALUES(p_school_id, p_amount, p_transaction_id, p_method, p_notes, NOW())
  RETURNING id INTO v_payment_id;
  
  -- Allocate to oldest unpaid years
  FOR v_year IN 
    SELECT * FROM school_billing_years 
    WHERE school_id = p_school_id 
      AND total_paid < annual_amount
    ORDER BY start_date ASC
  LOOP
    v_allocated := LEAST(v_remaining, v_year.annual_amount - v_year.total_paid);
    
    UPDATE school_billing_years 
    SET total_paid = total_paid + v_allocated
    WHERE id = v_year.id;
    
    INSERT INTO school_payment_allocations(payment_id, billing_year_id, allocated_amount)
    VALUES(v_payment_id, v_year.id, v_allocated);
    
    v_remaining := v_remaining - v_allocated;
    
    RETURN QUERY SELECT v_year.id, v_allocated;
  END LOOP;
  
  -- Advance balance if overpaid
  IF v_remaining > 0 THEN
    UPDATE school_billing_schedules
    SET advance_balance = advance_balance + v_remaining
    WHERE school_id = p_school_id;
  END IF;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;
```
